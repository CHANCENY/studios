Session creation endpoint.

To create session token you are required to call this endpoint as follows

You are required to sent a post request to /api/session/create with json body data with keys

                {
                  "username" : "chxxx@gmail.com",
                  "password" : "xxxxxxxxxxx"
                }

Note the username and password are the ones your have created using Create Account

In headers you only requires to set Content-Type value application/json

Sample code in Js XHR

                    // WARNING: For POST requests, body is set to null by browsers.
                    var data = JSON.stringify({
                         "username": "hhhhhn@gmail.com",
                         "password": "xxxxxxx"
                    });

                    var xhr = new XMLHttpRequest();
                    xhr.withCredentials = true;

                    xhr.addEventListener("readystatechange", function() {
                       if(this.readyState === 4) {
                          console.log(this.responseText);
                       }
                    });

                    xhr.open("POST", "https://api.streamstudios.online/api/session/create");
                    xhr.setRequestHeader("Content-Type", "application/json");
                    xhr.send(data);

Responses

If all goes well on your request you are expected to receive this response

                {
                  "status": 200,
                  "s-key": "b90e26a7c505180c2f5fc07bff2efc2860ca18cbc1dc26df2197ef2dbe719b10",
                  "msg": "You have successfully logged in. please always remember to set s-key header for all endpoints that requires s-key header thank you."
                }

If your request does not meet the requirement needed you are to get these responses

                Trying to login while already logged in

                {
                  "status": 404,
                  "link": "/api/tokens/token",
                  "msg": "You already logged in visit @link value endpoint to get your token if you lost it."
                }
                Sending request with method that is not POST

                {
                  "msg": "Bad request"
                }
                Sending request to this endpoint with token attached to s-key header

                {
                  "msg": "Trying to access endpoint only visible to anonymous users"
                }
