Movies listing endpoint.

To get movies listing you are required to call this endpoint as follows

You are required to sent a GET request to /api/movies/listing with no json body data

In headers you only requires to set Content-Type value application/json and s-key value current in use

Sample code in Js XHR

  
                    // WARNING: For GET requests, body is set to null by browsers.

                    var xhr = new XMLHttpRequest();
                    xhr.withCredentials = true;
                    
                    xhr.addEventListener("readystatechange", function() {
                      if(this.readyState === 4) {
                        console.log(this.responseText);
                      }
                    });
                    
                    xhr.open("GET", "https://api.streamstudios.online/api/movies/listing");
                    xhr.setRequestHeader("Content-Type", "application/json");
                    xhr.setRequestHeader("s-key", "e5721b5764f650261a1a15341010e6473733f29721de4b241a5032c14fab6aca");
                    
                    xhr.send();

Responses

If all goes well on your request you are expected to receive this response

                  {
                    "status": 200,
                    "results": [
                        {
                            "title": "Spy Kids: Armageddon",
                            "id": "b891952c-595f-11ee-b168-370b4a89ce3a",
                            "date": "2023-09-22",
                            "image": "https://streamstudios.online/sites/files/images/movies/spy-kid-armg.jpg"
                        },
        
                        more objects
                    ]    
                  }

If your request does not meet the requirement needed you are to get these responses

                  Trying to regenerate token with invalid token

                  {
                   "msg":"Missing header value for s-key or is invalid"
                  }
                  Sending request with method that is not PUT

                  {
                    "msg": "Bad request"
                  }

You can also set params of page number keyed page_number

Note Total movies per page its 12 movies

If happens that you have requested page that doesnt exist you will receive the error

            {
                "msg": "Pages ends at 40",
                "status": 404
            }
